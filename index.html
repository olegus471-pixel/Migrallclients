🔐 Архитектура API — данные через сервер Render

      
Все API-ключи хранятся в переменных среды Render (Environment Variables):
  GOOGLE_SHEETS_API_KEY   — ключ доступа к Google Sheets
  GOOGLE_SERVICE_ACCOUNT  — JSON сервис-аккаунта Google
  SHEETS_SPREADSHEET_ID_* — ID таблиц

CRM ↔ Backend (Render) ↔ Google Sheets

Потоки данных:
  GET  /api/clients          → читает из листа CRM_Clients
  POST /api/clients          → создаёт строку в листе CRM_Clients
  PUT  /api/clients/:id      → обновляет строку по ID
  GET  /api/statuses         → читает СПИСОК ПРОВЕРОК
  PUT  /api/statuses/:num    → обновляет статус + дату проверки
  GET  /api/courts           → читает лист судов
  PUT  /api/courts/:num      → обновляет статус суда
  GET  /api/schedule         → читает расписание
  POST /api/schedule         → добавляет слот
  GET  /api/visad            → читает статусы виз D
  PUT  /api/visad/:num       → обновляет статус визы
  PUT  /api/citizenship/:num → обновляет гражданство
  PUT  /api/rights/:num      → обновляет права

При изменении через CRM: данные → Render API → Google Sheets
При изменении в Sheets: Sheets Apps Script → POST /api/webhook → CRM обновляется
Календарь: отдельный поток через Google Calendar API (OAuth 2.0)

⚙️ Настройка Telegram

      

        Инструкция:

        1. Создайте бота через @BotFather → /newbot

        2. Подключите бота к бизнес-аккаунту: Telegram → Настройки → Telegram Business → Чат-боты

        3. Добавьте токен бота в Render: Environment Variables → TG_BOT_TOKEN

        4. Нажмите «Установить Webhook» ниже
